# Mytory i18n 에이전트 지침

이 폴더의 파일(`mytory-i18n.js` 등)을 수정한 후에는 반드시 테스트를 실행하여 기능의 무결성을 검증해야 합니다.

## 개발 원칙 (TDD)

본 프로젝트는 **테스트 주도 개발(TDD)**을 지향합니다.
1. 새로운 기능을 추가하거나 버그를 수정하기 전, 해당 케이스를 검증하는 테스트 코드를 `test.js`에 먼저 작성하십시오.
2. 테스트가 실패하는 것을 확인한 후, 코드를 수정하여 테스트를 통과시키십시오.
3. 수정한 후에는 전체 테스트(`npm test`)를 실행하여 회귀 버그가 없는지 확인하십시오.

## 테스트 실행 방법

```bash
# mytory-i18n 폴더에서 실행
npm install  # jsdom 등 의존성 설치가 필요한 경우
npm test
```

## 주요 테스트 항목

- **기본 작동**: 브라우저 언어 감지, 기본 언어 유지, HTML 보안 필터링(Sanitization), 부분 적용 기능.
- **v1.2.0 기능**: Locale 상세 매칭 및 폴백, `-attr-` 구분자를 이용한 속성 번역.
- **특수 기능**: URL 해시(`i18n-ko`)를 통한 강제 설정 및 실시간 변경 연동.
- **UI 제어**: `data-mi18n-block`(언어별 노출) 및 `data-mi18n-cloak`(초기 숨김) 로직.

## 주의 사항

- `jsdom` 라이브러리를 사용하여 브라우저 환경을 시뮬레이션합니다.
- 새로운 속성이나 옵션을 추가할 때는 `test.js`에 관련 테스트 케이스를 반드시 포함하십시오.
